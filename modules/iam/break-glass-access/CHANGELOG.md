# Changelog

## [v2.0] - 2026-07-01

### Added

- **Break-glass IAM user** (`iam.tf`) — Console-only IAM user in the management account with a random 32-character initial password (forced reset on first login), stored in SSM Parameter Store as a SecureString at `/break-glass/initial-password`.
- **Bookmarks S3 bucket** (`bookmarks.tf`) — Private, encrypted (AES-256), versioned bucket with TLS-only enforcement and a placeholder HTML object seeded on first deploy.
- **Refresh Lambda** (`lambda.tf`) — Python 3.14 Lambda (arm64) that enumerates all Organization accounts, determines Control Tower governance via `controltower:ListEnabledBaselines`, and renders a static switch-role HTML page to S3. Distinguishes CT-managed accounts (`AWSControlTowerExecution`) from non-CT accounts (`OrganizationAccountAccessRole`).
- **EventBridge lifecycle trigger** (`lambda.tf`) — Invokes the refresh Lambda on `CreateManagedAccount` / `UpdateManagedAccount` Control Tower events.
- **EventBridge schedule trigger** (`lambda.tf`) — Periodic refresh (default: weekly) to capture account suspensions that have no dedicated event.
- **Layered detective alerting** (`monitoring.tf`) — Three EventBridge rules with SNS email notifications:
  1. Console sign-in by the break-glass user.
  2. Cross-account role assumptions (AssumeRole / AssumeRoleWithSAML / AssumeRoleWithWebIdentity).
  3. Write (mutating) API calls (`readOnly = false`), excluding STS and noisy KMS crypto operations.
- **Conditional CloudTrail trail** (`monitoring.tf`) — When `enable_centralized_logging = false`, creates a minimal multi-region management-events-only trail to ensure EventBridge delivery; skipped when the CT organization trail is active.
- **Lambda source code** (`assets/lambda/index.py`) — Auto-discovers CT home region and baseline ID; resolves OU-level baselines to individual account IDs; renders a responsive HTML page with status badges and managed-by indicators.
- **`.gitignore`** — Excludes the `assets/lambda/build/` directory generated by `archive_file`.
- **`versions.tf`** — Requires Terraform ≥ 1.5.0, AWS provider ≥ 6.0.0, archive ≥ 2.0.0, random ≥ 3.0.0.

### Changed

- **Provider version constraint** bumped from AWS provider ≥ 5.x to ≥ 6.0.0 (breaking change for callers pinned to 5.x).

### Removed

- **`releases/v1.0.md`** — Previous release notes file removed from the module source tree (release docs now managed externally).
